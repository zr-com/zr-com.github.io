利用云平台搭建个人博客

实验目的

熟悉个人博客系统的搭建。注册Github账号，搭建Hexo环境并实现个人博客搭建，撰写实验报告。

硬件环境

联网的计算机一台

软件环境

Windows

实验步骤与内容

Github账号

搭建Hexo环境

1.标题配置本地 node 环境。node官网下载。

2.默认安装

3.环境配置

在你安装node的文件夹下新建两个文件夹；node_cache(缓存文件夹)node_global(全局仓库文件夹)

4.然后再命令行输入

npm config set prefix “node_global的地址”
npm config set cache “node_cache的地址”

5.修改

$npm config set registry http://registry.npm.taobao.org/ 修改淘宝源
$npm config set registry https://registry.npmjs.org/ 还原默认下载源

1.Git的安装

2.然后打开Git bash，输入 git –version查看 Git 版本号：

3.全局配置

安装完 Git 后还需要进行全局配置：

$ git config –global user.name “name”

$ git config –global user.email “email@example.com“

安装Hexo环境

1.$ npm install -g hexo-cli
$ hexo

显示下面情况，全局模块调用

2.$ hexo -v //查看版本

运行

1.$ mkdir hexo-blog
初始化hexo项目
在hexo-blog文件下初始化
$ cd hexo-blog
$ hexo init && npm install

2.最后提示我们可以启动博客了，我们使用命令 hexo server 或者简写为 hexo s：
$ hexo server
INFO Start processing
INFO Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

3.访问一下我们的4000端口：

部署至github page

1.创建github仓库

首先打开github，新建一个仓库，仓库名为：yourname.github.io，其中yourname就是你的github的用户名(昵称)。

2.安装部署 hexo 的插件：
$ cnpm install –save hexo-deployer-git

3.修改站点配置文件
然后我们打开第一个文件 _config.yml,这个文件是整个hexo博客的站点配置文件，想要部署到远端，我们需要对这个文件进行修改：

4.然后进行部署，一条命令就可 hexo -d：
$ hexo d
然后这个时候我们就可以使用 yourname.github.io 进行访问了。
用仓库名访问，和使用 localhost 进行访问的结果应该是一样的。

整体流程

发布文章在我的个人博客上

1.在csdn博客上或者是其他平台上用md格式编写你的文章。当然，直接使用markdown工具在本地编写也是可以的。

2.新建文档

在Hexo站点所在目录打开终端，在终端输入命令即可：


上述命令的结果是在 ./source/_posts 路径下新建了一个md 文件。

我们用typora打开该文件，会发现已经存在内容：


用于设置 MarkDown 文档在被解析为静态网页文件时的相关配置.
其中，title 的值是当前文档名，也是将来在网页中显示的文章标题。
date 值是我们新建文档时的当地时区时间。
tags 值是文档的标签，我们可以随意赋值为文档贴标签。

3.在你存放博客的blog文件夹目录下右击调出git，执行hexo g（generate）进行静态部署，然后输入hexo s（server）启动本地服务器，用于预览文章。最后执行hexo d（deploy)生成网页的静态文件，并部署到指定仓库。这样，一个博客就成功发布到你的网站上啦！
